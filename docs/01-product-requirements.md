# 白玩计划 - 产品需求文档（MVP，v0.5 修订版）

## 1. 产品概述

### 1.1 产品名称

**白玩计划**

### 1.2 产品定位

白玩计划是一款面向 NS1、NS2、PS5 游戏玩家的微信小程序，用于记录游戏的买入、持有、卖出和回血情况。

产品支持两类游戏记录：

1. **实体版游戏**：包括 NS 卡带、PS5 光盘等，可参与买入、卖出、回血、白玩指数、待回血提醒等账本统计。
2. **数字版游戏**：包括 Nintendo eShop、PlayStation Store 等购买的数字游戏，只作为游戏库记录和个人购买记录，不参与卖出回血、白玩指数、待回血提醒和买卖统计。

它不是普通的游戏收藏库，也不是严肃的金融资产工具，而是一个带有玩家趣味的游戏账本。

当前 UI 方向以移动端微信小程序为主，整体采用红白主色、轻游戏化、卡片化和清爽的账本总览风格。页面不做成游戏商城，也不做成金融资产看板。

核心表达：

> 记录你买实体游戏花了多少，卖游戏回了多少，以及哪些游戏实现了“白玩”。数字版游戏也可以记录，但不参与回血和白玩统计。

### 1.3 产品关键词

- 实体游戏
- 数字游戏
- 卡带 / 光盘
- 买入记录
- 卖出回血
- 白玩指数
- 白玩战绩
- 待回血
- 游戏账本
- 游戏库

## 2. 第一版产品边界

### 2.1 第一版做什么

MVP 版本主要完成：

1. 添加游戏记录
2. 区分实体版 / 数字版
3. 管理游戏库
4. 记录实体游戏买入花费
5. 记录实体游戏卖出回血
6. 计算实体游戏实际花费
7. 计算实体游戏白玩指数
8. 展示首页账本总览
9. 展示待回血游戏
10. 展示基础的最近记录
11. 首页提供快捷入口：登记买入、登记卖出、搜游戏库、待处理
12. 支持将收藏型游戏标记为“传家宝 / 收藏”，并排除在待回血提醒之外
13. 支持记录数字版游戏，但数字版不参与买卖统计、待回血提醒和白玩指数

### 2.2 第一版不做什么

第一版暂时不做：

1. 自动获取二手市场价格
2. 闲鱼 / 转转自动抓取价格
3. 当前市场估值
4. 未卖出游戏的预计盈亏
5. 价格趋势图
6. 社区价格库
7. OCR 识别截图价格
8. AI 识别游戏封面
9. 多端 App
10. 游戏社区功能
11. 扫码入库 / 条码识别
12. 数字版游戏的消费统计专项分析

### 2.3 重要说明

第一版没有二手价格数据，因此不展示以下指标：

- 当前估值
- 手里游戏大概值多少钱
- 未出售游戏预计赚亏
- 当前市场价
- 价格趋势
- 保值率排行榜

第一版首页账本统计只基于用户自己记录的**实体版游戏真实买卖数据**：

- 买入价格
- 买入运费
- 其他买入费用
- 卖出成交价
- 卖出运费
- 平台手续费
- 其他卖出费用

数字版游戏可以记录购买价格和购买日期，但默认不参与以下统计：

- 买游戏花了
- 卖游戏回血
- 实际花费
- 白玩指数
- 白玩战绩
- 待回血提醒
- 登记卖出

## 3. 目标用户

### 3.1 普通实体游戏玩家

特征：

- 会购买 NS / PS5 实体卡带或光盘
- 通关后可能会卖掉
- 想知道自己到底花了多少钱玩游戏
- 不一定追求赚钱，但希望少亏一点

核心需求：

> 我想记录自己买了哪些实体游戏，卖掉后回了多少钱。

### 3.2 二手交易型玩家

特征：

- 经常在闲鱼、转转、线下店买卖游戏
- 喜欢低价买入、高价卖出
- 对“回本”“白玩”“倒赚”比较敏感

核心需求：

> 我想知道哪些实体游戏最后接近白玩，哪些游戏卖得很划算。

### 3.3 收藏型玩家

特征：

- 喜欢收集实体游戏
- 可能不卖某些游戏
- 关心版本、区服、成色和收藏状态

MVP 对收藏型用户只提供基础支持：

- 可标记“收藏”
- UI 文案可显示为“传家宝”
- 收藏 / 传家宝游戏不进入待回血提醒
- 收藏 / 传家宝游戏不默认建议卖出

### 3.4 数字版游戏玩家

特征：

- 会购买 Nintendo eShop / PlayStation Store 数字版游戏
- 希望把数字版游戏也放进游戏库
- 不需要记录卖出，也不需要参与回血统计

MVP 对数字版用户只提供基础支持：

- 可添加数字版游戏
- 可记录购买价格、购买日期、购买渠道、服区和备注
- 可记录游玩状态
- 不参与卖出、回血、白玩指数和待回血统计

## 4. 核心用户路径

### 4.1 买入路径：实体版

```text
用户买入实体游戏
↓
打开白玩计划
↓
添加游戏
↓
选择游戏类型：实体版
↓
填写游戏名称、平台、买入价格、购买日期
↓
游戏进入游戏库，并参与首页账本统计
```

### 4.2 买入路径：数字版

```text
用户购买数字版游戏
↓
打开白玩计划
↓
添加游戏
↓
选择游戏类型：数字版
↓
填写游戏名称、平台、购买价格、购买日期、购买渠道
↓
游戏进入游戏库，但不参与卖出回血和白玩统计
```

### 4.3 持有路径

```text
用户游玩游戏
↓
更新游戏状态
↓
实体版状态可切换为：未开始、游玩中、已通关、已弃坑、待回血、出售中、收藏（UI 可显示为“传家宝”）
↓
数字版状态可切换为：未开始、游玩中、已通关、已弃坑、收藏
↓
系统根据实体版状态生成待回血提醒
```

### 4.4 卖出路径

```text
用户卖出实体游戏
↓
记录卖出成交价、运费、手续费
↓
系统计算卖出回血
↓
系统计算白玩指数
↓
生成卖出结果反馈
```

说明：

> 数字版游戏不支持卖出路径，不展示“记录卖出”入口，不生成白玩指数。

### 4.5 首页查看路径

```text
用户打开首页
↓
查看实体游戏买入总花费
↓
查看实体游戏卖出回血金额
↓
查看实体游戏实际花费
↓
查看白玩战绩
↓
查看待回血提醒
```

## 5. 底部 Tab 结构

MVP 使用 4 个底部 Tab：

```text
首页｜游戏库｜待回血｜我的
```

### 5.1 首页

作用：

- 展示实体游戏总账
- 展示白玩战绩
- 展示待回血提醒
- 提供醒目的添加游戏入口
- 展示最近记录

### 5.2 游戏库

作用：

- 管理所有游戏，包括实体版和数字版
- 搜索、筛选、排序
- 查看游戏详情
- 编辑游戏信息
- 为实体版游戏记录卖出

### 5.3 待回血

作用：

- 集中展示可以考虑处理的实体版游戏
- 包括已通关、已弃坑、待回血、出售中、长期未玩的实体版游戏
- 不展示数字版游戏

### 5.4 我的

作用：

- 设置
- 数据导出
- 平台管理
- 渠道管理
- 关于产品
- 后续账号相关功能

## 6. 游戏类型与状态设计

### 6.1 游戏类型

游戏类型字段为 `mediaType`：

| 类型   | 数据值   | 说明                                 | 是否参与买卖统计 |
| ------ | -------- | ------------------------------------ | ---------------- |
| 实体版 | physical | NS 卡带、PS5 光盘等实体游戏          | 是               |
| 数字版 | digital  | eShop / PlayStation Store 等数字游戏 | 否               |

### 6.2 实体版状态列表

实体版游戏状态包括：

| 状态   | 说明                                              |
| ------ | ------------------------------------------------- |
| 未开始 | 买了但还没开始玩                                  |
| 游玩中 | 正在玩                                            |
| 已通关 | 已完成主线或核心内容                              |
| 已弃坑 | 不打算继续玩                                      |
| 待回血 | 用户明确标记准备卖                                |
| 出售中 | 已经挂出或正在交易                                |
| 已出售 | 已完成卖出                                        |
| 收藏   | 标记为收藏，UI 可显示为“传家宝”，不参与待回血提醒 |

### 6.3 数字版状态列表

数字版游戏状态包括：

| 状态   | 说明                 |
| ------ | -------------------- |
| 未开始 | 买了但还没开始玩     |
| 游玩中 | 正在玩               |
| 已通关 | 已完成主线或核心内容 |
| 已弃坑 | 不打算继续玩         |
| 收藏   | 标记为收藏或保留     |

数字版不提供以下状态：

- 待回血
- 出售中
- 已出售

### 6.4 状态对待回血的影响

进入待回血提醒的条件：

1. 游戏类型 = 实体版
2. 同时满足以下任一状态或条件：
   - 状态 = 已通关
   - 状态 = 已弃坑
   - 状态 = 待回血
   - 状态 = 出售中
   - 买入超过 90 天且状态 = 未开始

不进入待回血提醒：

1. 游戏类型 = 数字版
2. 状态 = 已出售
3. 状态 = 收藏（传家宝）
4. 状态 = 游玩中

## 7. 核心指标定义

### 7.1 统计范围说明

以下核心账本指标默认只统计**实体版游戏**：

- 买入实际花费
- 卖出实际回血
- 实际花费
- 已卖出赚亏
- 白玩指数
- 白玩战绩
- 待回血数量

数字版游戏的购买金额仅作为游戏库信息保留，不计入首页账本统计。

### 7.2 买入实际花费

```text
买入实际花费 = 买入价格 + 买入运费 + 其他买入费用
```

说明：

> 仅实体版游戏参与首页“买游戏花了”统计。

### 7.3 卖出实际回血

```text
卖出实际回血 = 卖出成交价 - 卖出运费 - 平台手续费 - 其他卖出费用
```

说明：

> 仅实体版已出售游戏可计算卖出实际回血。

### 7.4 实际花费

用于首页总账：

```text
实际花费 = 实体游戏买入总花费 - 实体游戏卖出总回血
```

说明：

> 扣除卖游戏回血后，用户目前真正花在实体游戏上的钱。

### 7.5 已卖出赚亏

```text
已卖出赚亏 = 已卖出实体游戏的卖出实际回血总和 - 已卖出实体游戏的买入实际花费总和
```

### 7.6 最终花费

针对单个已出售实体游戏：

```text
最终花费 = 买入实际花费 - 卖出实际回血
```

如果最终花费大于 0：

```text
最终花了 ￥xx
```

如果最终花费等于 0：

```text
0 成本玩完
```

如果最终花费小于 0：

```text
倒赚 ￥xx
```

## 8. 白玩指数

### 8.1 白玩指数定位

白玩指数是白玩计划的特色玩法指标。

它不是严肃金融指标，而是一个带有玩家趣味的结果反馈，用来衡量一款实体游戏最终玩得是否划算。

### 8.2 计算公式

```text
白玩指数 = 卖出实际回血 / 买入实际花费 × 100
```

### 8.3 适用范围

白玩指数仅适用于：

- 游戏类型 = 实体版
- 状态 = 已出售
- 已记录买入实际花费
- 已记录卖出实际回血

数字版游戏不生成白玩指数。

### 8.4 解释文案

```text
100 表示基本回本，超过 100 表示倒赚。
指数越高，说明这款实体游戏玩得越划算。
```

### 8.5 是否封顶

白玩指数不封顶。

示例：

| 买入实际花费 | 卖出实际回血 | 白玩指数 |
| -----------: | -----------: | -------: |
|        ￥260 |        ￥260 |      100 |
|        ￥260 |        ￥250 |       96 |
|        ￥260 |        ￥285 |      110 |

### 8.6 等级设计

|  白玩指数 | 等级名称 | 说明                 |
| --------: | -------- | -------------------- |
|      120+ | 血赚玩家 | 玩到了，还赚了一笔   |
| 110 - 119 | 神级白玩 | 非常划算，明显倒赚   |
| 100 - 109 | 倒赚游玩 | 不但回本，还多回了钱 |
|   95 - 99 | 几乎白玩 | 最后只花了一点点     |
|   80 - 94 | 小亏体验 | 花小钱体验完整游戏   |
|   60 - 79 | 正常回血 | 卖出后回了一部分     |
|    0 - 59 | 回血失败 | 这次回血不太理想     |

### 8.7 白玩指数展示规则

已出售实体游戏：

- 展示正式白玩指数
- 展示等级名称
- 展示最终花费或倒赚金额

未出售实体游戏：

- 默认不展示正式白玩指数
- 展示“卖出后生成白玩指数”
- 如果后续支持挂出价 / 预期价，可展示“预计白玩指数”，但 MVP 不做重点

数字版游戏：

- 不展示白玩指数
- 不展示“卖出后生成白玩指数”
- 可展示“数字版游戏不参与白玩统计”

## 9. 首页需求

首页详见 `02-homepage-design.md`。本节补充当前 UI 已确认的首页信息结构。

首页核心模块：

1. 顶部标题区
2. 核心账本卡片
3. 快捷操作区
4. 最近入库
5. 白玩战绩
6. 待回血 / 待处理提醒
7. 最近记录

### 9.1 顶部标题区

内容：

- 标题：白玩计划
- 副标题：记录买入、卖出和白玩战绩
- 搜索入口

视觉要求：

- 标题需要更像品牌字标，不只是普通文本
- 红色 Hero 背景可以有轻量游戏元素，例如手柄轮廓、卡带轮廓、点阵或星芒
- 装饰元素要克制，不能做成商城 Banner
- 搜索入口保留在右上角

### 9.2 核心账本卡片

首页核心指标：

- 实体游戏实际花费
- 实体游戏买入花费
- 实体游戏卖出回血
- 当前持有实体游戏
- 回血率
- 距离完全白玩还差多少

说明：

- “实际花费”是首页最重要的核心数字
- 核心账本卡片默认只统计实体版游戏
- 回血率基于实体游戏买入实际花费和卖出实际回血计算
- MVP 不展示当前估值、市场价、未出售游戏预计盈亏
- 数字版游戏不进入核心账本卡片统计，可在游戏库中单独查看

### 9.3 快捷操作区

首页快捷操作区使用 4 个入口：

| 入口     | 说明                                               |
| -------- | -------------------------------------------------- |
| 登记买入 | 添加新买入的游戏，默认实体版，也可选择数字版       |
| 登记卖出 | 为已有实体版游戏记录卖出成交信息                   |
| 搜游戏库 | 搜索 / 筛选已添加或待添加的游戏                    |
| 待处理   | 查看待回血、出售中、已通关、已弃坑等可处理实体游戏 |

说明：

- MVP 不做扫码入库，因此不展示“扫码入库”入口
- “登记买入”比“登记录入”更明确
- “待处理”用于替代原“扫码入库”的位置
- 四个图标应保持统一风格，不要使用与功能不匹配的相机 / 扫码图标

### 9.4 最近入库

最近入库展示最近添加的游戏记录，包括实体版和数字版。

每个游戏卡片展示：

- 游戏封面
- 游戏类型标签：实体版 / 数字版
- 平台标签
- 游戏名称
- 买入实际花费或购买价格
- 当前状态
- 辅助说明，例如“入库 12 天”“挂牌中”“永久收藏 / 不卖出”“数字版 / 不参与回血”

如果游戏状态为收藏，卡片上可显示“传家宝”视觉标识。

### 9.5 传家宝 / 收藏展示规则

“传家宝”是“收藏”状态在 UI 层的趣味表达。

展示规则：

- 数据状态仍为 `collection`
- UI 展示文案可使用“传家宝”
- 最近入库封面上可显示轻量“传家宝”小标识
- 标识应像 App 原生 UI 状态，不要使用白底贴图、厚重金属章或促销贴纸
- 收藏 / 传家宝游戏不进入待回血提醒

### 9.6 白玩战绩

展示：

- 已卖出实体游戏
- 几乎白玩
- 倒赚游玩
- 最高白玩指数
- 收藏 / 传家宝数量可作为辅助信息展示

说明：

> 白玩战绩只统计实体版已出售游戏。数字版游戏不参与白玩战绩。

### 9.7 待回血 / 待处理提醒

首页可展示待处理摘要，例如：

- 已通关，可以考虑卖掉回血
- 已弃坑，可以考虑处理
- 出售中，记得记录成交
- 买入超过 90 天且未开始

收藏 / 传家宝游戏不进入该提醒。

数字版游戏不进入该提醒。

## 15. 数据模型初稿

### 15.1 GameItem

```ts
type GameItem = {
  id: string
  userId: string

  name: string
  mediaType: 'physical' | 'digital'
  platform: 'NS1' | 'NS2' | 'PS5' | 'OTHER'
  coverUrl?: string
  region?: string
  edition?: string
  condition?: string

  status:
    | 'not_started'
    | 'playing'
    | 'finished'
    | 'abandoned'
    | 'to_payback'
    | 'selling'
    | 'sold'
    | 'collection'

  purchasePrice: number
  purchaseShippingFee?: number
  purchaseOtherFee?: number
  purchaseDate: string
  purchaseChannel?: string

  soldPrice?: number
  sellShippingFee?: number
  sellPlatformFee?: number
  sellOtherFee?: number
  soldDate?: string
  sellChannel?: string

  note?: string
  createdAt: string
  updatedAt: string
}
```

### 15.2 mediaType 说明

`mediaType = 'physical'` 表示实体版游戏，参与买卖统计、卖出回血、白玩指数、待回血提醒。

`mediaType = 'digital'` 表示数字版游戏，只参与游戏库记录，不参与买卖统计、卖出回血、白玩指数、待回血提醒。

### 15.3 状态与 UI 文案说明

`status = 'collection'` 在数据层表示收藏状态。UI 层可以展示为“收藏”或“传家宝”。

MVP 建议：

```text
数据状态：collection
首页视觉文案：传家宝
说明文案：永久收藏 / 不卖出
```

如果后续需要同时记录“已通关 + 传家宝”，可以再扩展独立字段，例如 `isCollection` 或 `isHeirloom`。MVP 阶段为了保持模型简单，暂时沿用 `status = collection`。

### 15.4 数字版状态限制

当 `mediaType = 'digital'` 时，不允许使用以下状态：

- to_payback
- selling
- sold

如果用户从实体版切换为数字版，系统应提示并自动清理不适用的卖出字段。

### 15.5 ActivityLog

```ts
type ActivityLog = {
  id: string
  userId: string
  gameItemId?: string
  type: 'create' | 'update' | 'purchase' | 'sell' | 'status_change'
  title: string
  amount?: number
  createdAt: string
}
```

数字版游戏的添加也可以生成 ActivityLog，但不生成 sell 类型记录。
